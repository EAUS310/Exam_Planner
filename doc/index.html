FOE Exam Planner

A fully static exam scheduling and seat-assignment tool. No login, no installation — runs entirely in the browser.

What it does

Features

Exam Management

Create, edit, and delete exam entries with full multi-venue support.

Student Import

Paste raw student data (ID + Name) and auto-parse it instantly.

Automatic Seat Assignment

Alphabetical seat allocation across one or more venues, conflict-aware.

Manual Seat Editing

Override any student's assigned seat with duplicate-conflict checking.

Seating Optimisation

Interleaves students from different modules column-by-column to reduce copying risk.

Invigilator Scheduling

Assign and track invigilators per exam entry.

Shared Modules

Detect and manage modules that span multiple exams or venues.

Shared Venues

Track venue usage, detect scheduling conflicts, and import venue data.

Presentation Schedule

Log project presentation slots with per-instructor overlap detection.

Student ID Lookup

Search any student ID to see all their exam entries and seat allocations.

Exam Schedule View

Program-grouped schedule with filters and an A3 landscape print layout.

Print Outputs

Attendance sheets (with signature column) and seating plans (grouped by venue).

Quick Start

Option A — Live App (no setup)

Open the deployed app directly in your browser:

https://eaus310.github.io/Exam_Planner/

Option B — Open the HTML file locally

Open frontend/index.html directly in Chrome or Edge. All features work except file sync requires the Express server.

Option C — Express dev server

npm install
npm start        # starts at http://localhost:3000
npm run dev      # same but auto-restarts on file changes
How to use

Usage Guide

Creating an Exam

  1. Click "+ Add Exam Entry" on the main page.
  2. Fill in the details — module code, name, date, start/end time, instructor.
  3. Select one or more venues from the checklist.
  4. Click "Create Exam".

Adding Students

  1. Click "Students" next to any exam.
  2. Paste student data in the textarea — one per line, ID and name separated by tab or spaces: 
    2024001    Ali Hassan Mohammed
2024002    Fatima Al-Zahra Ahmed
  3. Click "Parse & Preview" to review parsed data.
  4. Click "Save Students & Assign Seats" — seats are auto-assigned alphabetically by last name.

Editing a Seat

  1. On the Student Management page, click "Edit Seat" next to a student.
  2. Enter the new seat label (e.g. B4).
  3. Click "Update Seat" — the system checks for duplicate seats before saving.

Seating Optimisation

Available when multiple exams share the same venue and time slot:

  1. Open the Student Management page for any of the shared exams.
  2. Click "Optimise Seating for All Exams in This Venue".
  3. Students from different modules are interleaved column-by-column so no two adjacent students share the same exam paper.

Printing

  1. Click "Print" next to an exam on the main page.
  2. Choose Attendance Sheet (sorted by seat, with signature column) or Seating Plan (grouped by venue).
  3. Use Ctrl+P to print or save as PDF.
Data

Data Storage

All data is stored in the browser's localStorage. Nothing is sent to a server — all exam and student data stays on the device unless you explicitly export or sync it.

File Sync (optional)

Click "Connect exams.json" in the sidebar to link a local file. Once linked, every change is automatically written back to that file via the File System Access API.

Browser requirement File sync requires Chrome or Edge. Firefox and Safari fall back to manual Import / Export JSON buttons.

Venues & Classrooms

Venue definitions live in frontend/data/classrooms.json and are loaded at runtime by storage.js → getClassrooms(). Two formats are supported — uniform (same rows per column) and irregular (each column declares its own row list).
